Introduction

Context

  • Second-Hand Book Market: Student and academic communities have numerous individual sellers operating small-scale book businesses
  • Delivery Challenge: Sellers must deliver books to multiple customers located at different addresses across the city
  • Manual Planning: Without proper tools, route planning is done manually, resulting in inefficient routes that waste time and fuel
  • Data Entry Burden: Sellers must manually copy each customer's delivery address from order records into navigation apps like Google Maps or Waze
  • Lack of Integration: Existing route optimization tools are separate from marketplace platforms, creating workflow friction

Problem Statement

Manual Address Handling
Sellers must manually copy each customer's delivery address from order records into navigation applications - a time-consuming and error-prone process
Inefficient Routes
Without optimization tools, sellers visit customers in order received or based on rough geographic estimates - rarely producing efficient routes
Lack of Integration
Route optimization tools are separate from marketplace platforms - sellers must use multiple disconnected applications, duplicating effort
Scalability Issues
Manual planning becomes impossible with 10+ daily deliveries - limiting business growth

Proposed Solution

  • Integrated Platform: Combines e-commerce marketplace functionality with automatic delivery route optimization
  • One-Time Address Entry: Customers register and provide delivery address once - automatically attached to all future orders
  • Mobile Route Planner: Sellers use mobile app to view pending orders, select which to deliver, and get optimal route sequence
  • TSP Algorithm: Traveling Salesman Problem algorithm calculates the best visit order to minimize travel distance
  • Turn-by-Turn Navigation: Integration with OpenRouteService, Google Maps, and Waze for detailed driving directions
  • Seamless Data Flow: Address data flows automatically from registration → order → route planning without manual entry

How Address Data Flows

Step 1: Registration
User registers on web platform and provides delivery address → stored in User entity in MongoDB
Step 2: Order Creation
When user places order, system automatically copies their address from User profile to Order entity
Step 3: API Retrieval
Seller opens mobile app → retrieves orders via API → each order already contains delivery address
Step 4: Route Planning
Seller selects orders → app extracts addresses → feeds into TSP algorithm for optimization
No manual address entry required at any step!

Route Optimization Algorithm

TSP Nearest-Neighbor

Problem Type
Traveling Salesman Problem (TSP) - finding optimal delivery sequence
Algorithm
Greedy Nearest-Neighbor Heuristic
Complexity
O(n²) time where n = number of delivery points
Solution Quality
Provides good (not perfect) solutions - acceptable for 5-20 delivery points

Algorithm Steps

1. START at seller's current GPS location 2. WHILE unvisited deliveries remain: a. Find nearest unvisited customer (using distance calculation) b. Add customer to route c. Mark as visited d. Move to that location 3. RETURN optimized route sequence 4. Send waypoints to OpenRouteService for turn-by-turn directions
Performance: Fast execution on mobile devices - optimizes 10 points in < 2 seconds
